# Polymarket Bot - Роадмап и Архитектура

## Основная стратегия
Бот на Python для Polymarket. Стратегия: микро-ставки (от $0.001 до $0.05 за долю) на маловероятные события ("чёрные лебеди") в **низколиквидных нишах**, где ценообразование наименее эффективно. Цель — высокая оборачиваемость капитала (~100-200 сделок/день), а не единичные крупные ставки. В случае редких выигрышей — от 10x до 1000x+ ROI.

### Подтверждение стратегии (реальный кейс)
Трейдер **planktonXD** (Polymarket, февраль 2025 — февраль 2026): из ~$1,000 стартового капитала через 72,533 автоматизированных предсказаний получил **$98,000+ прибыли**. Крупнейший одиночный выигрыш — $2,527 (лишь 2% от общей прибыли). Профит-кривая — стабильные 45° вверх без значимых просадок. Примеры: 3,664 акции победы Fuego по **0.1 цента** → +$874 (+23,750%); $16 на падение SOL до $130 по **0.7 цента** → +$1,574 (+9,285%).

### Ключевые ниши (высокая информационная асимметрия)
- **Киберспорт** — региональные лиги, мелкие турниры, раунды внутри матчей
- **Региональная политика** — выборы в малых странах, местные референдумы
- **Крипто-ценовые триггеры** — "достигнет ли X цены Y в период Z"
- **Погода и гиперлокальные события** — малоизвестные рынки с минимальной аудиторией

## Роадмап (План разработки)

**Этап 1. Подготовка и аналитика (Бэктестинг)**
- Изучение API Polymarket: Gamma API и CLOB API. Смотри [API_LIMITS.md](./API_LIMITS.md) для деталей по лимитам запросов.
- Сбор исторических данных по закрытым рынкам: любой длительности, у которых до закрытия оставалось **<= 48 часов** (охватывает как короткие рынки, так и длинные в финальной фазе). Детали архитектуры сбора описаны в [DATA_COLLECTION.md](./DATA_COLLECTION.md).
- Создание базы данных SQLite (Data Lake / Data Mart) с оптимизацией через LLM-суммаризацию для удешевления аналитики. Схема БД: [DATABASE_SCHEMA.md](./DATABASE_SCHEMA.md).
- Формализация критериев отбора: какие рынки берем, диапазон цен для ставки.

**Этап 2. Скринер рынков (Market Scanner)**
- *См. детали в [TRADING_LOGIC.md](./TRADING_LOGIC.md) и [RUNTIME_FLOW.md](./RUNTIME_FLOW.md)*
- Использование **WebSocket (Market Channel)** для минимизации задержек (Latency) и моментальной реакции на падение цен.
- Поиск токенов, цена которых падает в целевой диапазон (< $0.03 — приоритет на ультра-дешёвые, где асимметрия максимальна).
- **Приоритет низколиквидных рынков:** Скринер ранжирует рынки по признаку "низкая ликвидность + экстремально низкая цена" — именно там planktonXD находил 10,000%+ возможности.

**Этап 3. Исполнение ордеров (Execution)**
- *См. детали в [TRADING_LOGIC.md](./TRADING_LOGIC.md)*
- Использование **лимитных ордеров** или проверка глубины стакана для исключения проскальзывания (Slippage).
- Интеграция с кошельком и авторизация в CLOB API. Автоматический расчет Taker Fees.

**Этап 4. Управление позициями и мониторинг**
- *См. детали в [TRADING_LOGIC.md](./TRADING_LOGIC.md)*
- **Обязательный механизм Take Profit:** Сразу после покупки доли (например, за $0.02) выставляется лимитный ордер на продажу с целевым ROI (например, $0.20 для 10x). Если рынок закрывается до срабатывания TP, позиция рассчитывается по финальной цене.
- Система логирования и уведомлений (отчеты в Telegram). *См. детали в [LOGGING_AND_ALERTS.md](./LOGGING_AND_ALERTS.md)*.

**Этап 5. Тестирование и деплой**
- Режим "Dry Run" (симуляция торгов без реальных денег с Эмулятором Исполнения). *См. [TRADING_LOGIC.md](./TRADING_LOGIC.md)*.
- Запуск на небольшом депозите.
- Деплой на сервер (VPS) для круглосуточной работы.

## Архитектура (Структура проекта)
*Используемые библиотеки и готовые решения описаны в [BEST_PRACTICES.md](./BEST_PRACTICES.md).*

- `config.py` — настройки стратегии.
- `data_collector/` — скрипты для парсинга истории и работы с БД.
- `api/`
  - `gamma_client.py` — работа с публичным API.
  - `clob_client.py` — работа с биржевым стаканом.
- `strategy/`
  - `screener.py` — фильтрация рынков и поиск подходящих условий.
  - `niche_ranker.py` — ранжирование рынков по признаку информационной асимметрии (категория, ликвидность, аудитория).
  - `risk_manager.py` — расчет сайзинга (микро-позиции $0.001-$0.05), логика Take Profit.
- `execution/`
  - `order_manager.py` — формирование и отправка транзакций (защита от slippage).
- `utils/`
  - `logger.py` & `notifier.py` — логи и алерты.
- `main.py` — точка входа.
