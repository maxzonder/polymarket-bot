# Polymarket Bot - Роадмап и Архитектура

## Основная стратегия
Бот на Python для Polymarket. Стратегия: микро-ставки (от $0.001 до $0.05 за долю) на маловероятные события ("чёрные лебеди") в **низколиквидных нишах**, где ценообразование наименее эффективно. Цель — высокая оборачиваемость капитала (~100-200 сделок/день), а не единичные крупные ставки. В случае редких выигрышей — от 10x до 1000x+ ROI.

### Подтверждение стратегии (реальный кейс)
Трейдер **planktonXD** (Polymarket, февраль 2025 — февраль 2026): из ~$1,000 стартового капитала через 72,533 автоматизированных предсказаний получил **$98,000+ прибыли**. Крупнейший одиночный выигрыш — $2,527 (лишь 2% от общей прибыли). Профит-кривая — стабильные 45° вверх без значимых просадок. Примеры: 3,664 акции победы Fuego по **0.1 цента** → +$874 (+23,750%); $16 на падение SOL до $130 по **0.7 цента** → +$1,574 (+9,285%).

### Ключевые ниши (высокая информационная асимметрия)
- **Киберспорт** — региональные лиги, мелкие турниры, раунды внутри матчей
- **Региональная политика** — выборы в малых странах, местные референдумы
- **Крипто-ценовые триггеры** — "достигнет ли X цены Y в период Z"
- **Погода и гиперлокальные события** — малоизвестные рынки с минимальной аудиторией

## Роадмап (План разработки)

**Этап 1. Подготовка и аналитика (Бэктестинг)**
- Изучение API Polymarket: Gamma API и CLOB API. Рабочая выжимка: [API_REFERENCE.md](./API_REFERENCE.md). Rate limits и throttling: [API_LIMITS.md](./API_LIMITS.md).
- Сбор исторических данных по закрытым рынкам **без жёстких фильтров** — данные собираются полностью, пороги определяются аналитикой. Детали: [DATA_COLLECTION.md](./DATA_COLLECTION.md). Схема БД: [DATABASE_SCHEMA.md](./DATABASE_SCHEMA.md).
- Цель этапа: эмпирически найти оптимальный порог входа, временное окно и категории с положительным EV. Константы выводятся из данных, не задаются заранее.

**Этап 2. Скринер рынков (Market Scanner)**
- *Детали архитектуры: [TRADING_LOGIC.md](./TRADING_LOGIC.md) и [RUNTIME_FLOW.md](./RUNTIME_FLOW.md)*
- Screener работает через **REST polling раз в 5-10 минут** (Gamma API). Стратегия НЕ требует HFT — ищем рынки где цена уже находится на низком уровне, а не ловим момент падения.
- Скоринг каждого рынка по динамическим весам (категория, ликвидность, срок до закрытия). Веса обновляются Profit Module.
- WebSocket используется **только** для мониторинга собственных открытых позиций (TP-ордера).

**Этап 3. Исполнение ордеров (Execution)**
- *Детали: [TRADING_LOGIC.md](./TRADING_LOGIC.md). Готовые решения: [BEST_PRACTICES.md](./BEST_PRACTICES.md).*
- Лимитный ордер на покупку через `py-clob-client`. Сразу после исполнения — лимитный TP SELL.
- Проверка глубины стакана перед покупкой (`GET /book`). Интеграция с кошельком через CLOB API.

**Этап 4. Мониторинг позиций и Profit Module**
- *Детали: [TRADING_LOGIC.md](./TRADING_LOGIC.md), [RUNTIME_FLOW.md](./RUNTIME_FLOW.md)*
- **Position Monitor:** REST polling `GET /orders` раз в 1-2 минуты. Отслеживает статус TP: `ORDER_STATUS_MATCHED` (прибыль) или `ORDER_STATUS_CANCELED_MARKET_RESOLVED` (исход рынка).
- **Profit Module:** при каждом закрытии позиции пересчитывает EV-веса в `category_ev_stats`. Самообучение на реальных данных.
- Логирование и алерты в Telegram: [LOGGING_AND_ALERTS.md](./LOGGING_AND_ALERTS.md).

**Этап 5. Тестирование и деплой**
- Режим **Dry Run** — весь цикл включая Profit Module без реального USDC. *Детали: [TRADING_LOGIC.md](./TRADING_LOGIC.md).*
- Запуск на небольшом депозите после успешного Dry Run.
- Деплой на VPS для круглосуточной работы.

## Архитектура (Структура проекта)
*Используемые библиотеки и готовые решения: [BEST_PRACTICES.md](./BEST_PRACTICES.md). API-справочник: [API_REFERENCE.md](./API_REFERENCE.md).*

- `config.py` — настройки стратегии.
- `data_collector/` — скрипты для парсинга истории и работы с БД.
- `api/`
  - `gamma_client.py` — работа с публичным API.
  - `clob_client.py` — работа с биржевым стаканом.
- `strategy/`
  - `screener.py` — фильтрация рынков и поиск подходящих условий.
  - `niche_ranker.py` — ранжирование рынков по признаку информационной асимметрии (категория, ликвидность, аудитория).
  - `risk_manager.py` — расчет сайзинга (микро-позиции $0.001-$0.05), логика Take Profit.
- `execution/`
  - `order_manager.py` — формирование и отправка транзакций (защита от slippage).
- `utils/`
  - `logger.py` & `notifier.py` — логи и алерты.
- `main.py` — точка входа.
