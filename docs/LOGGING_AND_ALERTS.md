# Система Логирования и Оповещений (Logging & Alerts)

Для обеспечения прозрачности работы асинхронных модулей и быстрого реагирования на нештатные ситуации внедряется централизованная система логирования.

## 1. Логирование (`utils/logger.py`)

Вместо стандартного `print()` все модули должны использовать единую обертку над встроенной библиотекой `logging` Python.

### Требования к логгеру:
- **Формат логов:** `Дата Время | УРОВЕНЬ | Имя Модуля | Сообщение` (например: `2026-03-12 19:56:44 | INFO | MarketDiscovery | Спарсено 300 новых рынков`). Обязательное указание `__name__` модуля решает проблему путаницы в асинхронном выводе.
- **Dual Output:** Логи должны выводиться одновременно в терминал (stdout) и в файл.
- **Ротация лог-файлов:** Чтобы логи не забили память сервера, используется `RotatingFileHandler`. Максимальный размер файла — 5 МБ. Хранится история из последних 3-х бэкапов, старые удаляются.
- **Уровни:**
  - `DEBUG` — техническая инфа (json-ответы API, сырые цены). Включено только при локальной отладке.
  - `INFO` — штатная работа (найден кандидат, ордер выставлен).
  - `WARNING` — некритичные проблемы (лимиты Cloudflare близко, потерян сокет — идет реконнект).
  - `ERROR` / `CRITICAL` — ошибки исполнения, нет баланса, сбои базы данных.

## 2. Оповещения в Telegram (`utils/notifier.py`)

Критически важная информация не должна оставаться только в файле. Бот должен проактивно пушить её владельцу через Telegram (или другой мессенджер) через Webhook / Bot API.

### Что отправляется в Telegram:
1. **Торговые события (Успехи):** 
   - Покупка: *"🦢 Пойман лебедь! Куплено 100 долей $YES по $0.03 на рынке 'Will BTC hit 100k'. Выставлен TP на $0.30."*
   - Срабатывание Take Profit: *"💰 Сработал TP! Продано по $0.30. Прибыль: +$27."*
   - Закрытие рынка (Разрешение): *"🏁 Рынок закрыт. Токен проиграл (списано $3)."*
2. **Алерты об ошибках (Errors):**
   - Закончился свободный USDC баланс на кошельке.
   - Ошибка подписи транзакции (отказ CLOB API).
   - Глубокие технические сбои (скрипт упал с Exception).
3. **Ежедневный отчет (Daily Report):** 
   - Раз в сутки (например, в 00:00 UTC) бот присылает короткую сводку PnL (Profit and Loss), количество активных рынков в Watchlist-е и текущий баланс на Polygon.
